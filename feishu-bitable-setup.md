# 飞书多维表格搭建清单

这份清单按飞书多维表格的实际使用习惯来写，目标是让你第一次搭就能跑起来，而不是追求过度自动化。

## 一、表结构

建议建两张主表：

1. `商家总览表`
2. `拍摄批次表`

看板视图开在 `拍摄批次表` 上。

## 二、商家总览表

### 字段配置

| 字段名 | 类型 | 是否手填 | 用途 |
| --- | --- | --- | --- |
| 商家名称 | 单行文本 | 是 | 商家主名称 |
| 发片规则类型 | 单选 | 是 | 选项：单数日 / 双数日 / 自定义周几 |
| 发片周几 | 多选 | 是 | 仅 `自定义周几` 使用 |
| 当前可发库存 | 数字 | 是 | 当前还能直接发出的成片条数 |
| 每次拍摄产出 | 数字 | 是 | 默认填 `3` |
| 下一个发布日期 | 日期 | 是 | 最近一次需要发片的日期 |
| 未来第1次发布日期 | 日期 | 是 | 通常与下一个发布日期一致 |
| 未来第2次发布日期 | 日期 | 是 | 辅助判断库存支撑日期 |
| 未来第3次发布日期 | 日期 | 是 | 辅助判断库存支撑日期 |
| 未来第4次发布日期 | 日期 | 是 | 辅助判断库存支撑日期 |
| 未来第5次发布日期 | 日期 | 是 | 辅助判断库存支撑日期 |
| 库存可支撑到 | 公式/日期 | 否 | 当前库存最后一次能正常发片的日期 |
| 断更发布日期 | 公式/日期 | 否 | 下一次会断更的发布日期 |
| 断更前最后安全发布日期 | 公式/日期 | 否 | 与 `库存可支撑到` 一致，便于看板展示 |
| 最晚开始剪辑日 | 公式/日期 | 否 | 按断更日倒推 1 天 |
| 最晚拍摄日 | 公式/日期 | 否 | 按断更日倒推 2 天 |
| 最晚写脚本日 | 公式/日期 | 否 | 按断更日倒推 3 天 |
| 风险等级 | 公式/单选 | 否 | 红 / 黄 / 绿 |
| 备注 | 多行文本 | 是 | 特殊说明 |

### 商家总览表的核心口径

- `当前可发库存` 只统计已经剪好、今天就能发的成片
- `未来第1次发布日期` 到 `未来第5次发布日期` 先用辅助字段维护
- `库存可支撑到` 取“与库存条数对应的那个未来发布日期”
- `断更发布日期` 取“库存用完后的下一次发布日期”

### 公式逻辑

如果你的飞书环境支持标准公式字段，推荐这样配：

#### 1. 库存可支撑到

按库存条数取对应未来发布日期：

```text
IF(
  当前可发库存 = 1, 未来第1次发布日期,
  IF(
    当前可发库存 = 2, 未来第2次发布日期,
    IF(
      当前可发库存 = 3, 未来第3次发布日期,
      IF(
        当前可发库存 = 4, 未来第4次发布日期,
        IF(
          当前可发库存 >= 5, 未来第5次发布日期,
          BLANK()
        )
      )
    )
  )
)
```

#### 2. 断更发布日期

```text
IF(
  当前可发库存 = 0, 未来第1次发布日期,
  IF(
    当前可发库存 = 1, 未来第2次发布日期,
    IF(
      当前可发库存 = 2, 未来第3次发布日期,
      IF(
        当前可发库存 = 3, 未来第4次发布日期,
        IF(
          当前可发库存 = 4, 未来第5次发布日期,
          BLANK()
        )
      )
    )
  )
)
```

当库存大于 5 时，第一版建议你把 `未来第6次发布日期`、`未来第7次发布日期` 补上，或者直接人工确认一次，不要强行把公式写得太重。

#### 3. 倒推日期

```text
最晚开始剪辑日 = 断更发布日期 - 1天
最晚拍摄日 = 断更发布日期 - 2天
最晚写脚本日 = 断更发布日期 - 3天
```

如果飞书日期公式你不想折腾，最稳妥的方式是：

- 用日期差函数或日期偏移函数直接减天数
- 如果你的表里不好写公式，就先手动做成公式列复制

#### 4. 风险等级

按 `断更发布日期` 距离今天的天数来判定：

- `<= 3天`：红色
- `4-7天`：黄色
- `> 7天`：绿色

你也可以改成更保守的阈值：

- `<= 5天` 红色
- `6-10天` 黄色
- `> 10天` 绿色

### 未来发布日期的维护建议

这是第一版最关键的落地策略：不要一上来追求极重公式，先确保你每天能用。

#### 对 `单数日` / `双数日` 商家

推荐手动维护 `下一个发布日期`，再按 `+2天` 连续填充后续日期：

- 单数日：17、19、21、23、25...
- 双数日：18、20、22、24、26...

因为你的商家数量目前很少，这一步每家只花十几秒，稳定且不容易出错。

#### 对 `自定义周几` 商家

比如 `西施竹韵` 是：周一 / 周三 / 周五 / 周六 / 周日

建议每周一花 1 分钟手动补 5 个未来发布日期，比如：

- 2026-04-17
- 2026-04-18
- 2026-04-19
- 2026-04-20
- 2026-04-22

这就是“辅助字段 + 人工确认”的核心做法。第一版简单、稳、可查。

### 妙颜示例

以 `2026-04-16` 为例：

- 商家：妙颜
- 规则：双数日
- 当前可发库存：3
- 下一个发布日期：2026-04-18
- 未来第2次：2026-04-20
- 未来第3次：2026-04-22
- 未来第4次：2026-04-24

则：

- `库存可支撑到 = 2026-04-22`
- `断更发布日期 = 2026-04-24`
- `最晚写脚本日 = 2026-04-21`
- `最晚拍摄日 = 2026-04-22`
- `最晚开始剪辑日 = 2026-04-23`

## 三、拍摄批次表

### 字段配置

| 字段名 | 类型 | 是否手填 | 用途 |
| --- | --- | --- | --- |
| 批次名称 | 单行文本 | 是 | 例如 `妙颜-2026-04-24` |
| 所属商家 | 关联记录 | 是 | 关联到商家总览表 |
| 计划拍摄条数 | 数字 | 是 | 默认填 `3` |
| 目标接上的发布日期 | 日期 | 是 | 这一批内容要接上的发布日期 |
| 最晚写脚本日 | 日期 | 是 | 通常从商家总览表抄过来 |
| 最晚拍摄日 | 日期 | 是 | 通常从商家总览表抄过来 |
| 最晚开始剪辑日 | 日期 | 是 | 通常从商家总览表抄过来 |
| 脚本状态 | 单选 | 是 | 未开始 / 进行中 / 已完成 |
| 拍摄状态 | 单选 | 是 | 未开始 / 已排期 / 已完成 |
| 剪辑状态 | 单选 | 是 | 未开始 / 进行中 / 已完成 |
| 当前阶段 | 单选 | 是 | 待写脚本 / 待拍摄 / 剪辑中 / 待交付 / 已完成 / 已逾期 |
| 是否逾期 | 公式/复选框 | 否 | 是否超过当前阶段的最晚日期 |
| 备注 | 多行文本 | 是 | 特殊情况备注 |

### 批次表使用规则

- 当某个商家 `风险等级` 变黄或变红时，立刻新建一个批次
- `目标接上的发布日期` 填商家总览里的 `断更发布日期`
- `最晚写脚本日 / 最晚拍摄日 / 最晚开始剪辑日` 从商家总览复制过来
- 完成一批交付后：
  - 商家总览表里把 `当前可发库存 +3`
  - 批次表里把当前批次改成 `已完成`

### 当前阶段判定建议

第一版建议手动选，不用自动公式。

这样最省心，也最符合你日常做事的节奏：

- 还没写本：`待写脚本`
- 本子写完等去店里：`待拍摄`
- 素材拍完在剪：`剪辑中`
- 成片导出待发客户：`待交付`
- 这批已经交付：`已完成`
- 已经过了关键日期还没推进：`已逾期`

## 四、看板与视图

### 1. 拍摄批次表看板

按 `当前阶段` 分组：

- 待写脚本
- 待拍摄
- 剪辑中
- 待交付
- 已完成
- 已逾期

看板卡片上建议展示：

- 批次名称
- 所属商家
- 目标接上的发布日期
- 最晚写脚本日
- 最晚拍摄日
- 最晚开始剪辑日

### 2. 建三个常用视图

#### 本周必须处理

筛选条件建议：

- `目标接上的发布日期` 在本周内
  或
- `最晚写脚本日` 在本周内

#### 48小时内要行动

筛选条件建议：

- `最晚写脚本日 <= 今天+2`
  或
- `最晚拍摄日 <= 今天+2`
  或
- `最晚开始剪辑日 <= 今天+2`

#### 已逾期

筛选条件建议：

- `当前阶段 != 已完成`
- 且 `今天 > 最晚写脚本日 / 最晚拍摄日 / 最晚开始剪辑日` 中对应阶段日期

如果公式不好配，第一版直接手动把 `当前阶段` 切到 `已逾期` 也完全可接受。

## 五、条件格式建议

### 商家总览表

- `风险等级 = 红`：整行红底
- `风险等级 = 黄`：整行黄底
- `风险等级 = 绿`：整行正常

### 拍摄批次表

- `当前阶段 = 已逾期`：整行红底
- `目标接上的发布日期` 在 3 天内：橙色
- `当前阶段 = 已完成`：灰色

## 七、推荐日常操作流程

### 每天早上

1. 打开 `商家总览表`
2. 先看 `风险等级` 是红或黄的商家
3. 如果某家已经进入黄/红，马上确认是否建新批次

### 每次建新批次

1. 在 `拍摄批次表` 新增一行
2. `批次名称` 用 `商家名-目标接上日`
3. 把商家总览的 3 个最晚日期复制进去
4. 当前阶段先设为 `待写脚本`

### 每次交付 3 条后

1. 在商家总览把 `当前可发库存 +3`
2. 重填未来 5 次发布日期
3. 检查 `库存可支撑到`、`断更发布日期` 是否刷新
4. 把本次拍摄批次改成 `已完成`

## 八、第一版边界

这套设计故意保留一点人工维护，原因很明确：

- 你现在商家数量不多
- 发片规则并不统一
- 多维表格一开始公式写太重，后续反而难维护

先把这版跑顺最重要。等你商家到 10 家以上，再考虑：

- 新增 `视频成片明细表`
- 自动汇总库存
- 自动消息提醒
- 自动生成批次任务
