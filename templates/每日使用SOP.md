# 每日使用 SOP

## 每天早上先看什么

1. 打开 `商家总览表`
2. 先看 `风险等级`
3. 优先处理红色，其次处理黄色
4. 看 `断更发布日期` 和 `最晚写脚本日`

## 什么情况下要立刻新建拍摄批次

- 商家进入黄色或红色风险
- 商家 `最晚写脚本日` 已经到今天
- 你知道这家库存只够再发 1 到 2 次

## 新建拍摄批次时怎么填

- `批次名称`：商家名 + 目标接上日
- `所属商家`：选对应商家
- `计划拍摄条数`：默认填 `3`
- `目标接上的发布日期`：填商家总览里的 `断更发布日期`
- `最晚写脚本日 / 最晚拍摄日 / 最晚开始剪辑日`：直接从商家总览复制
- `当前阶段`：先选 `待写脚本`

## 推进阶段时怎么切

- 开始写本：还是 `待写脚本`
- 本子写完，准备去拍：改成 `待拍摄`
- 已拍完，开始剪：改成 `剪辑中`
- 剪好待交：改成 `待交付`
- 客户可正常发：改成 `已完成`
- 已经过了关键日期还没做：改成 `已逾期`

## 交付完 3 条后怎么维护

1. 回到 `商家总览表`
2. 把该商家的 `当前可发库存 +3`
3. 确认 `未来第1次发布日期` 到 `未来第5次发布日期`
4. 看 `库存可支撑到` 和 `断更发布日期` 有没有变化
5. 回到 `拍摄批次表`，把该批次改成 `已完成`

## 每周固定维护动作

每周一建议做一次：

1. 检查所有商家的未来 5 次发布日期
2. 特别是 `西施竹韵` 这种按周几更新的商家
3. 看本周有没有需要提前写本或拍摄的批次
4. 清掉已经完成的旧批次，保留记录即可
